Form controls

Inputs with helper messaging and validation

Pair every field with a clear label, optional supporting text, and inline error handling. Focus rings come for free via the existing form-control utility.

Error copy should explain how to recover, not just what went wrong.

Buttons & links

Four action tiers with clear intent

Use the semantic button classes defined in theme.css to avoid rebuilding paddings, corners, or focus rings.

When a view only exposes one action, default to the primary button. Never place two primary buttons side-by-side.

Pair buttons with descriptive verbs (“Create listing”) and optional helper text below when destructive.

Link pattern

Inline links use text-[#3A2440] + underline on hover. Navigation pills keep uppercase text with nav-link styles.

Icon + text alignment

Wrap icons inside a flex container with gap-2 so the hit target grows with the label.

Layout

Spacing, grid, and elevation guidance

The product breathes thanks to generous white space and rounded corners. Anchor pages inside a max-width of 80rem, then center with mx-auto and responsive padding ( px-6 on mobile).

Stack content using multiples of 8px. Cards use gap-6 inside while sections rely on py-10 to match the hero rhythm.

Typography

Consistent scale for product UI and marketing blocks

Inter sits on the Tailwind sans stack. The base size is 16px (1rem) with a vertical rhythm of 1.625. Reduce optical clutter by limiting yourself to these headline tiers.

Tracking for eyebrows uses 0.3em. Tighten hero headings with -0.01em letter-spacing when they exceed 32 characters.
